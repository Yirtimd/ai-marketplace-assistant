# AI Marketplace Assistant — Архитектура системы

## Обзор проекта

AI Marketplace Assistant — это AI-платформа для автоматизации работы продавцов на маркетплейсах.

Система выступает как **AI-операционная система для управления магазином**.

Основные задачи системы:

* анализ продаж
* контроль остатков
* генерация карточек товаров
* ответы на отзывы
* массовое управление товарами
* автоматизация операционных процессов

Главная идея:

AI должен не только анализировать данные, но и **принимать решения и выполнять действия**.

---

# Общая архитектура системы

Система состоит из трёх основных слоёв:

Client Layer
Backend Layer
AI Workflow Layer

```
Пользователь
    │
    ▼
Desktop Application (Electron + React)
    │
    ▼
Backend API (FastAPI)
    │
    ▼
Orchestrator
    │
    ▼
LangGraph Workflows
    │
    ▼
Agents
    │
    ▼
Services
    │
    ▼
External APIs (Wildberries + AI сервисы)
```

---

# Client Layer (Клиентская часть)

Технологии:

* Electron
* React

Назначение:

Десктопное приложение является основным интерфейсом пользователя.

Функции:

* отображение аналитики
* управление товарами
* управление отзывами
* запуск AI функций
* просмотр уведомлений
* управление настройками

Клиентское приложение работает как **интерфейс и API-клиент**.

Вся бизнес-логика выполняется на сервере.

---

# Backend Layer (Серверная часть)

Основной стек:

Python
FastAPI
PostgreSQL
Redis
Celery

Основные задачи backend:

* предоставление API
* хранение данных
* планирование задач
* запуск AI workflow
* взаимодействие с маркетплейсами
* взаимодействие с AI сервисами

---

# AI Workflow Layer

Технология:

LangGraph

LangGraph используется для реализации **AI workflow и многошаговой логики агентов**.

Каждый сложный процесс реализуется как **граф выполнения задач**.

Пример workflow:

```
Получение данных
      ↓
Анализ
      ↓
Принятие решения
      ↓
Выполнение действия
```

LangGraph позволяет:

* управлять состоянием процесса
* строить сложные логические цепочки
* реализовывать ветвления решений
* реализовывать многошаговые AI процессы

---

# Основные AI агенты

Система использует специализированных агентов.

Агенты являются **stateless компонентами**, которые выполняют конкретные задачи.

---

## Analytics Agent

Задачи агента:

* анализ продаж
* анализ динамики
* выявление трендов
* анализ конкурентов

Входные данные:

* продажи
* цены
* остатки
* позиции в поиске

Результат:

* аналитический отчет
* рекомендации

---

## Inventory Agent

Задачи:

* контроль остатков
* расчет скорости продаж
* прогноз окончания товара

Результат:

* уведомления о низких остатках
* рекомендации по заказу товара

---

## Content Agent

Задачи:

* генерация описания товара
* генерация изображений
* генерация видео
* SEO оптимизация карточки

Используемые технологии:

LLM модели
модели генерации изображений
модели генерации видео

---

## Review Agent

Задачи:

* анализ отзывов
* определение тональности
* генерация ответа
* публикация ответа

---

## Pricing Agent

Задачи:

* анализ цен конкурентов
* оценка спроса
* рекомендации по изменению цены
* изменение цены товара

---

# Orchestrator

Orchestrator отвечает за запуск нужного workflow.

Примеры:

Пользователь спрашивает:

"Почему упали продажи?"

→ запускается `sales_analysis_workflow`

Пользователь выбирает:

"Создать карточку товара"

→ запускается `product_creation_workflow`

---

# Services Layer

Слой сервисов отвечает за взаимодействие с внешними системами.

Основные сервисы:

wb_service
ai_service
image_generation_service
video_generation_service

Задачи сервисов:

* взаимодействие с API маркетплейсов
* вызов AI моделей
* обработка внешних данных

---

# Система задач (Task System)

Система включает планировщик задач.

Задачи могут выполняться:

* по расписанию
* через заданный интервал
* по событию

Примеры задач:

* проверка остатков каждый час
* анализ продаж каждый день
* проверка цен конкурентов

Используемые технологии:

Celery
Redis

---

# Event System (Система событий)

Архитектура системы основана на событиях.

Примеры событий:

new_review
stock_low
sales_drop
price_change

События автоматически запускают workflow.

Пример:

```
event: new_review
↓
review_workflow
```
