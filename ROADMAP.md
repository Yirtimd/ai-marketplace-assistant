# ROADMAP.md

# План разработки проекта AI Marketplace Assistant

Этот файл описывает поэтапный план разработки системы.

Основная цель roadmap — разбить разработку на логические этапы, чтобы система развивалась последовательно и архитектура не разрушалась.

---

# Этап 1 — Базовая инфраструктура проекта

Цель этапа: создать техническую основу системы.

Задачи:

* создать структуру backend проекта
* настроить FastAPI
* подключить PostgreSQL
* подключить Redis
* настроить базовую систему логирования
* настроить систему конфигурации (env)

Структура backend на этом этапе:

```
backend/

api/
services/
database/
config/
```

Результат этапа:

* backend сервер запускается
* база данных подключена
* система готова к разработке бизнес-логики

---

# Этап 2 — Интеграция с Wildberries

Цель этапа: получить доступ к данным магазина.

Задачи:

* реализовать сервис для работы с Wildberries API
* получение списка товаров
* получение продаж
* получение остатков
* получение отзывов

Создается сервис:

```
services/wb_service.py
```

Функции сервиса:

* get_products()
* get_sales()
* get_stocks()
* get_reviews()

Результат этапа:

система может получать данные магазина.

---

# Этап 3 — Модель данных системы

Цель этапа: создать структуру базы данных.

Создаются основные таблицы.

Основные сущности:

users
shops
products
sales
reviews
tasks
events
workflow_state

Результат этапа:

система имеет устойчивую модель данных.

---

# Этап 4 — Task System и Scheduler

Цель этапа: реализовать систему задач.

Задачи:

* подключить Celery
* настроить Redis как очередь
* реализовать scheduler

Типы задач:

* периодические задачи
* отложенные задачи
* события

Примеры задач:

* анализ продаж раз в сутки
* проверка остатков каждый час
* мониторинг цен

Результат этапа:

система может выполнять автоматические задачи.

---

# Этап 5 — Orchestrator

Цель этапа: создать центральный механизм запуска workflow.

Создается компонент:

```
orchestrator/
orchestrator.py
```

Функции orchestrator:

* принимать запросы
* определять нужный workflow
* запускать workflow

Примеры:

запрос пользователя → запуск workflow
событие → запуск workflow

Результат этапа:

система может запускать AI процессы.

---

# Этап 6 — Базовая система агентов

Создаются основные агенты.

Структура:

```
agents/

analytics_agent.py
inventory_agent.py
review_agent.py
content_agent.py
pricing_agent.py
```

Каждый агент отвечает за конкретную область.

Агенты должны быть stateless.

Результат этапа:

архитектура становится модульной.

---

# Этап 7 — AI Workflows (LangGraph)

На этом этапе внедряется система workflow.

Создается папка:

```
workflows/
```

Создаются основные workflow.

sales_analysis_workflow
product_creation_workflow
review_workflow
inventory_workflow
pricing_workflow

Результат этапа:

AI может выполнять сложные многошаговые процессы.

# Этап 11 — Автоматизация

Финальный этап MVP.

AI начинает:

* анализировать продажи
* автоматически реагировать на события
* запускать workflow
* давать рекомендации

Примеры автоматизации:

* уведомления о падении продаж
* уведомления о низких остатках
* предложения изменить цену
