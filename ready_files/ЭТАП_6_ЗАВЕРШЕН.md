## Реализованные агенты

### 1) AnalyticsAgent

Назначение:
- анализ продаж
- выявление трендов
- сравнение цены с рынком

Основные методы:
- `analyze_sales()`
- `detect_trends()`
- `analyze_competitors()`

### 2) InventoryAgent

Назначение:
- расчет скорости продаж
- прогноз окончания остатков
- рекомендации по пополнению

Основные методы:
- `calculate_sales_velocity()`
- `predict_stock_out()`
- `generate_reorder_recommendation()`

### 3) ReviewAgent

Назначение:
- анализ тональности отзывов
- генерация текста ответа
- подготовка payload для публикации

Основные методы:
- `analyze_sentiment()`
- `generate_reply()`
- `publish_reply()`

### 4) ContentAgent

Назначение:
- генерация описаний товаров
- SEO оптимизация
- подготовка задач для генерации изображений и видео

Основные методы:
- `generate_product_description()`
- `generate_product_images()`
- `generate_product_video()`
- `optimize_seo()`

### 5) PricingAgent

Назначение:
- анализ цен конкурентов
- оценка спроса
- рекомендации по новой цене
- подготовка payload для изменения цены

Основные методы:
- `analyze_competitor_prices()`
- `estimate_demand()`
- `recommend_price()`
- `update_price()`

## Архитектурные принципы соблюдены

✅ Stateless-дизайн: агенты не хранят состояние  
✅ Разделение ответственности: каждый агент решает одну доменную область  
✅ Модульность: агенты вынесены в отдельный слой `agents/`  
✅ Интеграция через workflow: агенты подключаются внутри workflow, а не напрямую друг к другу  
✅ Без прямых внешних API вызовов внутри агентов (Action/API остаются в соответствующих слоях)
