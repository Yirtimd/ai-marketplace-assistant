### 4. CheckInventoryWorkflow (`workflows/check_inventory.py`)

Первый пример workflow на **LangGraph**.

**Архитектура workflow:**

```
load_inventory → check_stock → generate_recommendations → format_result → END
```

**LangGraph State:**
```python
class InventoryState(TypedDict):
    shop_id: int
    workflow_id: str
    threshold: int
    products: list
    low_stock_products: list
    recommendations: list
    result: dict
```

**Nodes (шаги):**
1. `load_inventory` — загрузка данных из БД (через Repository)
2. `check_stock` — проверка уровней остатков
3. `generate_recommendations` — генерация рекомендаций (rule-based)
4. `format_result` — форматирование результата

**Результат:**
```json
{
  "workflow_id": "...",
  "shop_id": 1,
  "total_products": 150,
  "low_stock_count": 12,
  "critical_count": 3,
  "high_priority_count": 5,
  "recommendations": [
    {
      "product_id": 123,
      "nm_id": 456789,
      "title": "Product Name",
      "current_stock": 0,
      "priority": "critical",
      "recommended_action": "Order immediately"
    }
  ],
  "status": "completed"
}
```

### 5. Интеграция с EventListener

EventListener теперь запускает workflows через Orchestrator.

**Пример:**
```python
# Event: LOW_STOCK
async def on_low_stock(self, event: Event):
    # Trigger Celery task
    task = check_stock_levels_task.delay(threshold=10)
    
    # Trigger AI workflow (NEW!)
    workflow_result = await self.orchestrator.execute_workflow(
        workflow_name="check_inventory",
        context={"shop_id": shop_id, "threshold": 10},
        shop_id=shop_id
    )
```

**Результат:** Event-driven архитектура полностью функциональна!

### 6. API Endpoints (`api/routes/workflows.py`)

**Endpoints:**

| Method | Endpoint | Описание |
|--------|----------|----------|
| POST | `/api/v1/workflows/execute` | Запустить любой workflow |
| GET | `/api/v1/workflows/{workflow_id}` | Статус workflow |
| GET | `/api/v1/workflows/` | Список доступных workflows |
| POST | `/api/v1/workflows/inventory/check` | Shortcut для check_inventory |

**Примеры использования:**

#### Запустить workflow:
```bash
curl -X POST http://localhost:8000/api/v1/workflows/execute \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_name": "check_inventory",
    "context": {"shop_id": 1, "threshold": 10},
    "shop_id": 1
  }'
```

**Response:**
```json
{
  "workflow_id": "abc123-...",
  "status": "completed",
  "result": {
    "total_products": 150,
    "low_stock_count": 12,
    "recommendations": [...]
  }
}
```

#### Получить статус:
```bash
curl http://localhost:8000/api/v1/workflows/abc123-...
```

#### Список workflows:
```bash
curl http://localhost:8000/api/v1/workflows/
```
